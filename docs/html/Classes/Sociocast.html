Overview

Sociocast objects are used to interact with the Sociocast REST API Endpoints using SCRequest objects. See http://www.sociocast.com/dev-center/ for more information on working with the Sociocast REST API.

Tasks

Properly Initializing a Sociocast Object

  • – initWithAPIKey:andSecret:andClientID:

    Initialize a Sociocast object with an APIKey, APISecret and ClientID. This needs to only be initialized one time in your application. APIKey, APISecret and ClientID are provided to you by your Sociocast Client Support Manager. Email support@sociocast.com and see http://www.sociocast.com/dev-center/ for more information.

Sociocast REST API Endpoints

  • – entityObserve:delegate:

    Triggers an Entity Event to be tracked by Sociocast. Corresponds with the entity/observe API endpoint.
  • – entityAttributes:delegate:

    Used to pass Entity Attributes to Sociocast (i.e. Attributes specific to an Entity as opposed to an event)
  • – entityProfile:delegate:

    Used to get Entity attributes and Entity predictions (i.e. “scored” attributes). These are the attributes that were passed in as Entity Attributes (/entity/attributes) or using Entity Observe (/entity/observe).
  • – contentProfile:delegate:

    Used to directly classify URLs into Sociocast Content Categorization Taxonomy classifications.

Sociocast Object Functionality

  • – requestWithPath:HTTPMethod:parameters:delegate:

    Create a Sociocast Request object SCRequest. Used in lieu of a specific Endpoint function.
  • – generateSHA256Signature:APISecret:timeStamp:

    Compute a signature used to sign every API request. Signature consists of a SHA256 Digest based on the following key/value string, sorted in alphabetical order: APIKey, APISecret and Unix Timestamp. Note that a +/-5 minutes wiggle period is permitted on the Sociocast server on either side of the current timestamp to allow for reasonable clock drift and other delays.

generateSHA256Signature:APISecret:timeStamp:

Compute a signature used to sign every API request. Signature consists of a SHA256 Digest based on the following key/value string, sorted in alphabetical order: APIKey, APISecret and Unix Timestamp. Note that a +/-5 minutes wiggle period is permitted on the Sociocast server on either side of the current timestamp to allow for reasonable clock drift and other delays.

- (NSString *)generateSHA256Signature:(NSString *)key APISecret:(NSString *)secret timeStamp:(NSString *)timeStamp

Parameters

key

Sociocast granted APIKey

secret

Sociocast granted APISecret

timeStamp

Reflects the number of seconds since the Unix Epoch (January 1 1970 00:00:00 GMT) at the time a request was made.

Return Value

A NSString containing a SHA-256 Signature used to sign and authenticate every SCRequest sent to the Sociocast REST API.

Discussion
